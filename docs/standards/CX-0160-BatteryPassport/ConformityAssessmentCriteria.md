
# Conformity Assessment Criteria (CaC) – CX-0160-BatteryPassport

The conformity assessment criteria (CACs) per use case are outlined in the following sections.

## Proof of Conformity for Use Case (1): PROVIDING BATTERY DATA TO THE ECONOMIC OPERATOR

The data models MUST be implemented as described in [Semantic Models](./CX-0160-BatteryPassport.md#3-semantic-models).
The process and business logic MUST be implemented as described in [5.1 Use Case (1): PROVIDING BATTERY DATA TO THE ECONOMIC OPERATOR](./CX-0160-BatteryPassport.md#51-use-case-1-providing-battery-data-to-the-economic-operator).
The section [2.1.1.1 Use Case (1) DIGITAL TWINS AND SPECIFIC ASSET IDs](./CX-0160-BatteryPassport.md#2111-use-case-1-digital-twins-and-specific-asset-ids) MUST be implemented by as described.

The use case 1 distinguishes between the role of the producer and the customer.
The customer MAY choose to not provide any data. In this case, no data models need to be registered or provided.

## Proof of Conformity for Use Case (3): Bridging CATENA-X DPP to EU DPP System

The data models MUST be implemented as described in [Semantic Models](./CX-0160-BatteryPassport.md#3-semantic-models).
The section [2.1.1.2 Use Case (3) DIGITAL TWINS AND SPECIFIC ASSET IDs](./CX-0160-BatteryPassport.md#2112-use-case-2-digital-twins-and-specific-asset-ids) MUST be implemented by as described.
